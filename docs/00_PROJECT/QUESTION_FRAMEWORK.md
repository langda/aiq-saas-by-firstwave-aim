# QUESTION_FRAMEWORK.md

# Assessment Philosophy

Questions must evaluate real workplace behavior.

Never ask: - What is prompt engineering? - What is GPT?

Always ask: - What would you do?

## Question Formula

Scenario ↓ Decision ↓ Trade-off ↓ Hidden competency mapping

## Dimensions

1.  Curiosity
2.  Problem Solving
3.  Decision Making
4.  Learning
5.  Efficiency
6.  Workflow Design
7.  Judgment
8.  Vision

## Rules

-   4 answer choices.
-   Every option must sound reasonable.
-   Best answer position changes every question.
-   Avoid jargon.
-   Everyday office scenarios.

## Hidden Scoring

Each option maps to one or more competencies.

Example

Option A Curiosity +2 Judgment +1

Option B Workflow +2 Learning +1

Do not expose weights to users.

## Results

Overall score: 0--100

Display: - Overall score - Persona - Competency chart - Strengths -
Blind spots - Three recommended actions

## Question Bank

Each question stores: - Title - Scenario - Options - Competencies -
Difficulty - Industry tags - Status - Version

Target: 100+ questions over time.
